以前学习过JAVA, 由于许久不动, 许多知识已经淡忘, 并且第一次并没有经过系统的学习,导致了许多方面都出现了知识漏洞, 因此再次学习JAVA, 并在此留下脚印.

STEP 1

1. 关于整型常量的默认类型

在JAVA中,整型常量默认的类型是 int ,理论上说 byte b = 100 这种表达式是有问题的,但是JAVA对这种情况进行了特殊处理,
如果数据大小没有超过 byte/short/char 的表示范围,则自动进行类型转换.

同样由于默认的整型常量是int类型的,因此声明 long 变量注意一点,
long a = 10000000000(此数据大于 int 的最大值)
会导致报错,正确的方式是在10000000000后边加上一个L,则不会引发报错
long a = 10000000000L;

2. 关于浮点类型

浮点数常量默认的类型为 double 类型
float a = 1.23;
将会产生报错,因此需要:
float a = 1.23f;

支持科学计数法 12e3表示12乘以10的3次方 12000

值得思考的问题:
float 类型占4个字节, long 类型占8个字节,为什么 float 表示的范围要比 long 大?

计算机组成原理:Cache--主存地址映射

发表于 2016-10-12   |   分类于 计算机组成原理   |  

由主存地址映射到Cache地址成为地址映射,地址映射方式有很多,有直接映射, 全相联映射, 组相联映射.

Cache的引入

通过大量经典程序的分析,发现CPU从主存取指令或取数据,在一定时间内,只是对主存局部地址区域访问. 由于指令和数据在主存中的地址分布不是随机的, 而是相对的簇聚, 使得CPU在执行程序时,访问具有相对的局部性,这就称为程序访问的局部性原理

汉明码传输纠错过程

奇偶校验介绍

汉明码具有一位纠错能力,在了解汉明码如何生成和纠错之前,需要了解奇偶校验问题.

奇偶校验是一种校验代码传输正确性的方法,根据被传输的一组二进制代码的数位中“1”的个数是奇数或偶数来进行校验。采用奇数的称为奇校验，反之，称为偶校验。采用何种校验是事先规定好的。通常专门设置一个奇偶校验位，用它使这组代码中“1”的个数为奇数或偶数。若用奇校验，则当接收端收到这组代码时，校验“1”的个数是否为奇数，从而确定传输代码的正确性。
