hexo + github 搭建博客

发表于 2016-08-09   |   分类于 博客建设   |  

前言

1. 目标

使用hexo和github搭建一个简易的博客,满足基本需求

2. 前提

前提: 
    1.拥有github账号,掌握github最基本的操作  
    2.有命令行工具cmd或者gitBash(推荐)

正文

1. 工具准备

  1. 下载git(会使用github的各位一定已经有这个工具了)
  2. 下载node.js 点击下载

2. 开始安装

  1. 安装git和下载好的nodejs
  2. 在某个盘符创建一个文件夹,随便起个名字,这将会是你博客所在的目录,可以叫做hexo或者blog什么的,清楚自己在做什么就行了.

  3. 右键打开gitBash,输入命令 

    npm install hexo -g

    开始安装hexo及其依赖的插件等,需要一段下载的时间,耐心等待

  4. 进入到博客目录内 以D:\hexoblog为例子,在该目录内右键打开gitbash,输入命令

    hexo init

    刚才的命令会使hexo在博客目录内生成必要的文件,稍等片刻,该命令执行完毕后,输入以下命令

    npm install

    将依赖包安装

  5. 生成静态网页文件,输入命令:

    hexo g

    启动服务器,输入命令:

    hexo s

    完成以后,就可以在本地访问你的博客了.

3. 预览更改

生成了网页文件并启动了服务器,如何预览我们刚刚做好的博客呢? 执行 hexo s以后,网站已经跑起来了,打开浏览器,复制地址http://localhost:4000/到地址栏,回车访问,就能看到漂亮的博客页面了!

在gitbash中,按Ctrl+C就可以停止运行服务器.

4.配置文件概览

在博客目录中,有两个需要熟悉的配置文件,一个是全局的配置文件,一个主题的配置文件,全局的配置文件就位于根目录中,文件名:_config.yml,下面介绍了配置文件中的主要参数作用:

# Hexo Configuration  
## Docs: http://hexo.io/docs/configuration.html  
## Source: https://github.com/hexojs/hexo/  

# Site    这下面的几项配置都很简单，你看我的博客就知道分别是什么意思
title: Chillax blog    #博客名
subtitle: Goals determine what you are going to be    #副标题
description: Goals determine what you are going to be #用于搜索，没有直观表现
author: huangjunhui    #作者
language: zh-CN    #语言
timezone:     #时区，此处不填写，hexo会以你目前电脑的时区为默认值

# URL    暂不配置，使用默认值
## If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/'
url: http://opiece.me  #域名
root: /
permalink: :year/:month/:day/:title/
permalink_defaults:

# Directory        暂不配置，使用默认值
source_dir: source
public_dir: public
tag_dir: tags
archive_dir: archives
category_dir: categories
code_dir: downloads/code
i18n_dir: :lang
skip_render:

# Writing    文章布局等，使用默认值
new_post_name: :title.md # File name of new posts
default_layout: post
titlecase: false # Transform title into titlecase
external_link: true # Open external links in new tab
filename_case: 0
render_drafts: false
post_asset_folder: false
relative_link: false
future: true
highlight:
  enable: true
  line_number: true
  tab_replace:

# Category & Tag    暂不配置，使用默认值
default_category: uncategorized
category_map:
tag_map:

# Date / Time format    时间格式，使用默认值
## Hexo uses Moment.js to parse and display date
## You can customize the date format as defined in
## http://momentjs.com/docs/#/displaying/format/
date_format: YYYY-MM-DD
time_format: HH:mm:ss

# Pagination    
## Set per_page to 0 to disable pagination
per_page: 10    #每页显示的文章数，0表示不分页
pagination_dir: page

# Extensions    插件配置，暂时不配置
## Plugins: http://hexo.io/plugins/
## Themes: http://hexo.io/themes/
plugins:
- hexo-generator-feed
theme: landspace    #使用的主题，即：E:\myblog\themes文件夹下的主题文件夹名

feed:    #之后配置rss会用，使用如下配置即可
  type: atom
  path: atom.xml
  limit: 20  

# Deployment    用于部署到github
## Docs: http://hexo.io/docs/deployment.html

deploy: 
  type:

最后一项配置用于部署到github,我们下边马上就要谈到.
接下来是主题对应的配置文件,如果你没有使用过其他主题,那么这个文件应该位于博客根目录\themes\landscape下,文件的名称和上边一样,同样大致的介绍一下 

menu:    #博客右上角的菜单栏，暂时使用默认值
  首页: /
  归档: /archives
  关于: /about

#该类对应于E:\hexo\themes\light\layout\_widget下的文件
widgets:    #站点右边栏，暂时使用默认值
- intro        #简介
- search        #搜索
- category    #分类
- tagcloud    #标签云
- weibo        #微博
- blogroll    #友情链接



excerpt_link: Read more    #文章下的Read more，可以改为'阅读全文'

plugins:    #插件，暂时使用默认值

twitter:    #twitter
  username:
  show_replies: false
  tweet_count: 5

addthis:    #SNS分享，暂时使用默认
  enable: true
  pubid:
  facebook: true
  twitter: true
  google: true
  pinterest: true

fancybox: true    #图片效果，使用默认值

google_analytics: 
rss:     #生成RSS，暂时使用默认值

你的配置文件和上边列出的可能有很大不同,你现在只需要大致了解配置内容即可,有兴趣可以查阅相关资料.

5. 部署到github

在本地折腾博客自然是没什么好玩的,github提供平的page功能可以将你的博客搭建在互联网上,分享你的知识和经历.

  1. 创建一个仓库,该仓库的名称应该是 yourname.github.io 的格式,例如我的仓库名称是smalldy.github.io

  2. 修改配置文件,打开全局配置文件,找到最后一项配置项 

    # Deployment    用于部署到github
## Docs: http://hexo.io/docs/deployment.html

deploy: 
  type:

    修改如下:

注意: yml语法很严格,修改配置项的时候一定要注意每一个冒号后边都有一个空格!
配置项中的repository:后的地址,是创建仓库后得到的https:地址,但在这里写成http就可以

  1. 博客目录下打开gitbash,输入命令hexo g,生成静态网页文件,hexo s预览一下当前博客的效果,满意的话hexo d 部署到github
  2. 通过xxx.github.io访问你的博客
  3. 如果你在操作中遇到报错,请输入命令npm install hexo-deployer-git --save后再次尝试

6. 更新博文

博客搭建完毕,只有不断的更新文章博客才具有价值,使用hexo new "title"来创建一篇新的文章,在\source\_post就可以看到一个以md为后缀的文件了,md文件时以markdown语言为标准的文本文件,具体百度之,这里就不做赘述,利用hexo搭建的博客,需要用markdown语言来书写文章,如你所见,这篇文章同样是使用markdown语言写的.
写完以后,hexo g重新生成静态文件,hexo s启动服务器预览效果 hexo d部署到github上

7. 更换主题

github上有许多hexo主题供你clone,你需要将他们clone到本地然后放到themes下的一个文件夹中,文件夹的名字是你自己决定的,一般用主题的名字作为文件夹的名字,如果你得到的主题已经是一个文件夹了,那么就不必费心了,此时找到全局配置文件,找到theme:,经原来的主题更改为想要使用的主题所在的文件夹的名称 g s 预览 d 部署,此时你就可以辛勤的耕耘啦~

8. 防坑指南

  1. 务必注意,配置文件每一个配置项之后的冒号后都要追加一个空格 注意缩进问题
  2. 仓库连接可以在该项目仓库首页的clone的向下的小箭头中找到,也可以自己加上.git
  3. 出现 at…. lin xxx 表示这是在报错,你的操作可能有问题
  4. 确保你的github或者其他远程仓库已经配置好ssh key
  5. 理解,hexo d 是在本地生成网页文件, hexo s是将本地网页文件部署到本主机4000端口上 hexo d 是将这些网页文件推送到远程仓库之中
  6. 如果hexo d报错了,但是你却不知道错在什么地方npm install hexo-deployer-git --save这条命令可能会帮助你.
